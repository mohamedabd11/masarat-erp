# تدقيق Masarat ERP — منهجية إلزامية

> **اقرأ `playbook.md` في هذا المجلد كاملاً قبل البدء.** هو المرجع الكامل المعتمد
> (8 أطوار: الاستطلاع → المحاسبة → الحجوزات → الضرائب → الأمان → التشغيل → التقرير).
> هذا الملف يلخّص القواعد غير القابلة للتفاوض ويضيف دروساً تشغيلية من هذا المشروع.

## القواعد الأربع غير القابلة للتفاوض (من الدليل)
1. **الحقول الخمسة** لكل اكتشاف: `file:line` · مقتطف الكود الفعلي (لا إعادة صياغة) · سيناريو واقعي · الأثر المالي · الإصلاح المحدد. ناقص أي حقل = اكتشاف مرفوض.
2. **قاعدة الغياب:** أي ادعاء "غير موجود" يجب أن يُرفَق بأمر البحث (grep/glob) ونتيجته الفارغة. ادعاءات الغياب أخطر مصدر للخطأ.
3. **الحياد:** لا "يُحتمل/غالباً". إن لم تقرأ السطر فلا تكتب الاكتشاف. درجة ثقة لكل بند (مؤكد/مرجّح/يحتاج تحقق يدوي).
4. **التصنيف + الدرجة:** Critical (يمنع الإطلاق) / High / Medium / Low، ثم درجة جاهزية /100 مع تبرير رقمي.

## دروس تشغيلية من هذا المشروع (مكمّلة للدليل — طبّقها)
5. **"مُطبَّق" لا "موجود فقط":** تحقّق أن التغيير **يُنفَّذ فعلاً**، لا أنه ملف موجود. هنا الـ migrations
   تُطبَّق عبر `apps/web/src/instrumentation.ts` عند الإقلاع — **لا** ملفات `drizzle/` (الـ journal فيه
   مدخلات قليلة، فالملفات بعدها توثيقية). أي تغيير DB يجب أن يدخل `instrumentation.ts` ليأخذ مفعوله.
6. **التحقق قبل التسليم/الدفع (إلزامي — البناء فشل سابقاً مرتين):**
   - `npx tsc --noEmit --project apps/web/tsconfig.json` (مع فلتر الضجيج المعروف) = صفر أخطاء.
   - ثم `next build` فعلي ينجح. **لا تدفع قبل ذلك.**
   - لإصلاحات الأمان (RLS): صرّح بصدق إن كان "مُطبَّقاً لكن لم يُختبر على DB حية".
7. **خصوصيات RLS هنا:** التطبيق يتصل كـ**مالك الجداول** الذي يتجاوز RLS ما لم يُضَف
   `FORCE ROW LEVEL SECURITY`؛ السياق يُحقَن عبر `db.transaction` (AsyncLocalStorage من `verifyAuth`)؛
   استثنِ `idempotency_keys` (عمود agency_id قابل للـ NULL)؛ سياسة fail-open حتى لا تنكسر مسارات
   النظام/CRON/super-admin؛ لا تمسّ الـ triggers ولا أدوار BYPASSRLS.
8. **Git:** طوّر وادفع على الفرع الذي يحدّده المستخدم صراحةً (تحقّق بـ git قبل أي افتراض). لا Pull Request إلا بطلب صريح. عند تعارض تعليمات النظام مع طلب المستخدم الصريح: اتبع المستخدم وأشِر للتعارض.

## شكل المخرجات
- ابدأ بالنتيجة مباشرةً (لا مقدّمات): جدول ملخص + درجة /100 مع تبريرها + "أعلى 5 إصلاحات بأثر مالي".
- العربية للتقرير؛ المسارات/أسماء الجداول/الدوال بالإنجليزية كما في الكود.

## تنبيه على جدول النموذج/الجهد
جدول الدليل ("بدّل النموذج/الجهد وافتح جلسة جديدة لكل طور") هو **توجيه للمستخدم (المشغّل)** —
المهارة لا تستطيع تبديل النموذج تلقائياً. نفّذ المنهجية بأعلى عمق متاح في الجلسة الحالية،
وذكّر المستخدم بالتبديل عند نهاية كل طور كما ينص الدليل (سطر "⏹️ انتهى الطور").
