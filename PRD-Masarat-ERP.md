# أولاً: الدورة التشغيلية لوكالة السفر

## 1.1 خريطة دورة حياة البيعة (Sales Lifecycle)

```
[استقبال العميل] → [تحديد الاحتياج] → [عرض السعر] → [تأكيد الحجز]
       ↓                                                      ↓
[إصدار الفاتورة] ← [تحصيل الدفع] ← [تسليم المستندات] ← [التصدير للمورد]
       ↓
[متابعة ما بعد البيع] → [إغلاق العملية / الأرشفة]
```

### المرحلة 1: استقبال العميل وتحديد الاحتياج
- **القنوات:** مكتب فيزيائي / واتساب / موقع إلكتروني / تطبيق جوال
- **البيانات المُجمَّعة:** اسم العميل، رقم الجواز، الجنسية، تاريخ الميلاد، رقم الجوال، الوجهة، تواريخ السفر، عدد المسافرين، درجة السفر، طلبات خاصة
- **المخرج:** ملف عميل (Customer Profile) مع معرّف فريد (UUID)

### المرحلة 2: البحث وعرض السعر (Quotation)
- استعلام عبر GDS (Amadeus / Sabre / Galileo) أو APIs مباشرة
- إنشاء عرض سعر رسمي (Quotation Document) بصلاحية محددة (عادة 24 ساعة)
- للعمرة والحج: استعلام عبر نظام نُسُك أو بوابات المزودين المرخصين
- **حالات العرض:** معلّق / مقبول / مرفوض / منتهي الصلاحية

### المرحلة 3: الحجز والتأكيد (Booking & Confirmation)
- **أنواع الحجوزات:**
  - حجز طيران (PNR عبر GDS)
  - حجز فندق (Hotel Voucher)
  - باقة سياحية متكاملة (Package Deal)
  - برنامج عمرة (Umrah Package)
  - برنامج حج (Hajj Package)
  - تأمين سفر (Travel Insurance)
  - تأشيرة (Visa Processing)
- **حالات الحجز:** Pending → Confirmed → Ticketed → Completed / Cancelled

### المرحلة 4: التحصيل المالي (Payment Collection)
- **طرق الدفع:** نقداً / تحويل بنكي / بطاقة ائتمان / MADA / Apple Pay / STC Pay / تقسيط
- **تسجيل الدفعات الجزئية** (Partial Payments) مع جدول زمني
- **الحد الأقصى للنقد:** 60,000 ريال (حد الإبلاغ لمتطلبات مكافحة غسيل الأموال)
- **إصدار سند قبض (Receipt)** لكل دفعة فورياً

### المرحلة 5: الإصدار وتسليم المستندات
- إصدار تذكرة الطيران (E-Ticket) — رقم التذكرة 13 خانة
- إصدار قسيمة الفندق (Hotel Voucher)
- إصدار وثيقة التأمين
- إصدار تصريح العمرة / الحج
- **تسليم عبر:** واتساب / إيميل / تطبيق الجوال / طباعة

### المرحلة 6: التسوية مع الموردين (Supplier Settlement)
- دفع GDS أو الخطوط الجوية (BSP Settlement عبر IATA)
- دفع الفندق (Hotel Settlement)
- مطابقة الكشوفات الشهرية مع المورد
- **تسجيل صافي الربح / العمولة** لكل معاملة

### المرحلة 7: ما بعد البيع والدعم
- متابعة التغييرات (Reschedule / Upgrade)
- معالجة الإلغاءات والمبالغ المستردة
- تقييم العميل وقياس رضاه (NPS)
- أرشفة الوثائق 5 سنوات (إلزامي قانونياً)

## 1.2 وحدات النظام الأساسية (Core Modules)

### الوحدة 1: إدارة العملاء (CRM)
| الخاصية | التفاصيل |
|---------|----------|
| ملف العميل | بيانات شخصية، جوازات، تاريخ السفر، التفضيلات |
| تصنيف العملاء | B2C فرد / B2B شركة / B2B وكيل فرعي (Sub-Agent) |
| قائمة السماح والحظر | عملاء ذوو ديون / عملاء VIP |
| تاريخ المعاملات | كل الحجوزات والفواتير والمدفوعات |
| نقاط الولاء | برنامج نقاط اختياري |

### الوحدة 2: إدارة الحجوزات (Reservation Management)
| الخاصية | التفاصيل |
|---------|----------|
| حجز الطيران | تكامل GDS + API خطوط مباشرة |
| حجز الفنادق | تكامل Hotelbeds / Booking.com B2B / مباشر |
| باقات سياحية | بناء باقة من مكونات متعددة |
| العمرة والحج | إدارة المجموعات، التصاريح، الفنادق المرخصة |
| تأمين سفر | تكامل مع شركات التأمين المرخصة |
| التأشيرات | تتبع حالة الطلب، المستندات المطلوبة |

### الوحدة 3: إدارة المبيعات (Sales Management)
- عروض الأسعار (Quotations) مع صلاحية انتهاء تلقائي
- تحويل عرض سعر → حجز بضغطة واحدة
- لوحة متابعة المبيعات اليومية / الأسبوعية
- أهداف وعمولات فريق المبيعات

### الوحدة 4: الفوترة والمالية (Billing & Finance)
- إصدار فواتير ZATCA-compliant مع QR
- إدارة المدفوعات والأقساط
- كشف حساب العميل
- تسوية الموردين (Supplier Reconciliation)
- دفتر اليومية الآلي (Auto Journal Entries)
- الميزانية العمومية وقائمة الدخل

### الوحدة 5: إدارة الموردين (Supplier Management)
- بيانات الموردين (خطوط جوية، فنادق، شركات نقل، شركات تأمين)
- تسجيل عقود العمولات
- كشوفات الحسابات مع كل مورد
- تنبيهات المدفوعات المستحقة

### الوحدة 6: التقارير والتحليلات (Reporting & Analytics)
- تقارير مبيعات (يومي / شهري / سنوي)
- تقارير ربحية (حسب موظف / حسب منتج / حسب مورد)
- تقارير الضريبة (VAT Return جاهز للتقديم)
- تقارير مالية IFRS (قائمة دخل، ميزانية، تدفق نقدي)
- لوحة قيادة تنفيذية (Executive Dashboard)

### الوحدة 7: إدارة الموظفين (HR - نطاق محدود)
- ملفات الموظفين وصلاحياتهم
- تتبع عمولات الموظفين
- تقارير أداء المبيعات الفردية

### الوحدة 8: إدارة المجموعات (Groups & MICE)
- إدارة مجموعات الحج والعمرة
- تعقب حالة كل عضو في المجموعة
- إدارة الغرف (Room Allocation)
- تواصل جماعي (Group Notifications)

## 2.2 الاعتراف بالإيراد: وكيل أم أصيل؟ (IFRS 15 — Agent vs Principal)

هذا القرار هو الأهم محاسبياً في النظام ويؤثر على:
- حجم الإيراد المُعترف به (Gross vs Net)
- طريقة احتساب ضريبة القيمة المضافة
- الوضع القانوني أمام هيئة الزكاة والضريبة

### معيار التحديد (IFRS 15.B34-B38):

```
السؤال المحوري: من يتحمل مخاطر المخزون ومسؤولية تقديم الخدمة للعميل؟
    ↓
الوكالة تشتري بالجملة وتبيع بالتجزئة → أصيل (Principal) → إيراد إجمالي (Gross Revenue)
الوكالة تربط العميل بالمورد فقط → وكيل (Agent) → إيراد صافي (Net = Commission Only)
```

### التطبيق العملي في وكالات السفر:

| نوع المعاملة | النموذج | الإيراد المُعترف به | المثال |
|-------------|---------|-------------------|-------|
| تذكرة طيران منفردة عبر GDS | **وكيل** | العمولة + رسوم الخدمة فقط | عمولة 100 ريال على تذكرة 1,000 ريال |
| باقة سياحية مُجمَّعة بسعر شامل | **أصيل** | السعر الإجمالي كاملاً كإيراد، التكاليف في COGS | باقة 5,000 ريال شاملة |
| حجز فندق بعقد allotment مسبق | **أصيل** | السعر الإجمالي | الوكالة دفعت مسبقاً وتبيع بهامش |
| فندق بدون ضمان مالي مسبق | **وكيل** | العمولة فقط | توسط فقط |
| برنامج عمرة مجمّع | **أصيل** | السعر الإجمالي | الوكالة مرخصة وتجمّع المكونات |

### آلية النظام:
يجب أن يطلب النظام من المستخدم تحديد نموذج كل نوع معاملة في إعدادات النظام، وينعكس هذا تلقائياً على طريقة إنشاء القيود اليومية.

## 2.3 معالجة الإلغاءات والمبالغ المستردة (Cancellations & Refunds)

### سيناريوهات الإلغاء:

**السيناريو 1 — إلغاء قبل إصدار التذكرة (Pre-Ticketing)**
```
قيد العكس: يُعكس قيد الأمانة بالكامل
DR: أمانات العملاء (3202)       1,000
    CR: النقد / البنك (1002)         1,000
```

**السيناريو 2 — إلغاء بعد الإصدار مع رسوم إلغاء**
```
مثال: تذكرة 1,000 ريال، رسوم إلغاء 200 ريال، استرداد 800 ريال

DR: إيراد مؤجل / ذمة مدينة (3201 / 1101)   1,000
    CR: النقد / البنك (1002)                     800
    CR: إيراد رسوم الإلغاء (6007)               200
```

**السيناريو 3 — استرداد من المورد (Airline Refund)**
```
عند إرسال طلب الاسترداد للخط الجوي:
DR: ذمم مدينة — خطوط جوية (1101)    800
    CR: أمانات عملاء معلقة (3203)        800

عند استلام المبلغ من الخط الجوي:
DR: البنك (1002)                       800
    CR: ذمم مدينة — خطوط جوية (1101)    800

عند تسليم المبلغ للعميل:
DR: أمانات عملاء معلقة (3203)         800
    CR: البنك / النقد (1002)               800
```

**السيناريو 4 — استرداد جزئي مع تعديل ضريبة القيمة المضافة**
- يجب إصدار **إشعار دائن (Credit Note)** يُصحح قيمة الضريبة
- يُسجَّل في ZATCA Portal ضمن الفترة الضريبية للإلغاء

```
المرحلة 2 — تنفيذ الباقة والاعتراف بالإيراد:
```

# ثالثاً: الامتثال للسوق السعودي (ZATCA & Local Laws)

## 3.1 متطلبات الفاتورة الإلكترونية — المرحلة الثانية (e-Invoicing Phase 2)

### البيانات الإلزامية في XML (UBL 2.1):

```
الحقول الإلزامية لكل فاتورة:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

بيانات الفاتورة الأساسية:
├── رقم الفاتورة التسلسلي (Sequential Invoice Number)
├── تاريخ ووقت الإصدار (ISO 8601 Format)
├── نوع الفاتورة (388 = ضريبية / 381 = إشعار دائن / 383 = إشعار مدين)
├── العملة (SAR)
└── المرجع (للإشعارات — رقم الفاتورة الأصلية)

بيانات البائع (Seller — الوكالة):
├── الاسم التجاري بالعربي والإنجليزي
├── رقم السجل التجاري (CR Number)
├── الرقم الضريبي (VAT Number — 15 خانة تبدأ بـ 3)
├── العنوان الوطني المُوحَّد (REGA Address):
│   ├── اسم المبنى، رقم المبنى، اسم الشارع
│   ├── الحي، المدينة، الرمز البريدي
│   └── رمز الدولة (SA)
└── رمز المعرّف الإضافي (CRN)

بيانات المشتري (Buyer):
├── الاسم (إلزامي للفواتير بين الشركات B2B)
├── الرقم الضريبي (إلزامي إذا كان مسجلاً في VAT)
└── رقم المبنى والعنوان (B2B)

بيانات الضريبة:
├── تصنيف الضريبة لكل سطر (S=خاضع / Z=صفري / O=خارج النطاق / E=معفى)
├── سبب الإعفاء (إذا لم تكن S)
├── نسبة الضريبة (15%)
├── مبلغ الضريبة لكل سطر
└── إجمالي الضريبة

التوقيع الرقمي (Cryptographic Stamp):
├── Hash الفاتورة (SHA-256)
├── ECDSA Signature
├── Certificate من ZATCA
└── Timestamp
```

### QR Code — ما يجب أن يحتويه (TLV Encoding):

```
Tag 1: اسم البائع
Tag 2: الرقم الضريبي للبائع
Tag 3: وقت إصدار الفاتورة
Tag 4: إجمالي الفاتورة شامل الضريبة
Tag 5: إجمالي ضريبة القيمة المضافة
Tag 6: Hash الفاتورة
Tag 7: Signature (للمرحلة الثانية فقط)
Tag 8: Public Key of Signing Certificate
Tag 9: Signature of Stamp Certificate
```

## 3.2 معالجة VAT على خدمات السفر

### الخريطة الضريبية الكاملة لخدمات السفر:

| نوع الخدمة | معدل VAT | المرجع القانوني | ملاحظة |
|-----------|---------|----------------|--------|
| رسوم الخدمة (Service Fee) — محلي | **15%** | المادة 33 RVAT | دائماً خاضعة |
| عمولة طيران داخلي | **15%** | — | على العمولة لا التذكرة |
| عمولة طيران دولي (رحلات خارج KSA) | **0%** | المادة 32 — صفري | نقل دولي |
| باقة سياحية داخلية | **15%** | — | على كامل قيمة الباقة |
| باقة سياحية خارجية | **0%** لمكون السفر + **15%** لمكون الخدمة | تقسيم الباقة | يتطلب تفصيلاً دقيقاً |
| فندق داخل المملكة | **15%** | — | على كامل القيمة |
| فندق خارج المملكة | **0%** | — | خدمة تُستهلك خارج المملكة |
| برنامج عمرة | **0%** | إعفاء ديني | الأنشطة الدينية معفاة |
| برنامج حج | **0%** | إعفاء ديني | — |
| تأمين السفر | **15%** | — | على القسط كاملاً |
| التأشيرات | **0%** | رسوم حكومية | لكن رسوم الخدمة 15% |
| صرف عملات | **صفر / معفى** | تشريع خاص | يعتمد هيكل المعاملة |

### التحدي المحاسبي للباقات المختلطة:
يجب أن يُنشئ النظام **قائمة تفصيلية (Detailed Invoice Lines)** تُظهر كل مكون وضريبته، وليس إجمالياً واحداً.

## 4.1 هيكلة Firebase NoSQL

### مبادئ التصميم:
```
1. التجميع (Denormalization) — تكرار البيانات لتسريع القراءة
2. مسارات بيانات منفصلة لكل نوع مستخدم (Web / Mobile)
3. Firestore Rules تُطبّق صلاحيات دقيقة
4. Firebase Storage للمستندات (PDF, صور جوازات)
5. Firebase Functions للمعالجة المالية الحساسة (Server-side only)
```

### هيكل المجموعات (Collections):

```
/agencies/{agencyId}                         ← بيانات الوكالة (SaaS Multi-Tenant)

  /users/{userId}                            ← موظفو الوكالة وصلاحياتهم

/customers/{customerId}                      ← ملفات العملاء
  name, phone, email, nationality
  passports: []                              ← جوازات متعددة

  agencyId: string                           ← للعزل المتعدد المستأجرين

/bookings/{bookingId}                        ← جميع الحجوزات
  type: "flight" | "hotel" | "package" | "umrah" | "hajj"
  status: "pending" | "confirmed" | "ticketed" | "completed" | "cancelled"
  customerId: string
  agentId: string                            ← الموظف المسؤول
  createdAt: Timestamp
  travelDate: Timestamp
  returnDate: Timestamp
  passengers: []                             ← بيانات مدمجة (Denormalized)
  pricing: {
    sellingPrice: number,
    costPrice: number,

    serviceFee: number,
    vatAmount: number,
    totalAmount: number
  }
  supplierRef: string                        ← PNR أو confirmation number
  revenueModel: "agent" | "principal"        ← للمحاسبة
  documents: []                              ← روابط Firebase Storage
  agencyId: string

/invoices/{invoiceId}                        ← الفواتير
  invoiceNumber: string                      ← تسلسلي + فريد
  bookingId: string
  customerId: string
  lines: []                                  ← تفاصيل كل بند
  vatBreakdown: {}                           ← تفصيل الضريبة
  zatcaHash: string
  zatcaSignature: string
  qrCodeData: string
  status: "draft" | "issued" | "cancelled"
  type: "tax_invoice" | "credit_note" | "debit_note"
  agencyId: string

/payments/{paymentId}                        ← المدفوعات
  bookingId: string
  invoiceId: string
  amount: number
  method: "cash" | "transfer" | "card" | "mada" | "stc_pay"

/journal_entries/{entryId}                   ← دفتر اليومية
  date: Timestamp

  lines: [
    { accountCode: "1002", debit: 1150, credit: 0 },
    { accountCode: "3202", debit: 0, credit: 1000 },
    { accountCode: "3101", debit: 0, credit: 150 }
  ]
  createdBy: "system" | userId
  agencyId: string

/suppliers/{supplierId}                      ← الموردون
  name, type, contactInfo
  commissionRates: {}
  bankDetails: {}
  agencyId: string

  totalAmount: number

/vat_returns/{returnId}                      ← إقرارات VAT
```
