# تشغيل بيانات السيناريوهات التجريبية — دليل المبتدئين

هذا الملف يشرح كيف تملأ حسابك في النظام ببيانات تجريبية حقيقية (فواتير، دفعات،
تقسيط، استرداد، إيراد مؤجل، دفعات موردين) عشان تتأكد بنفسك أن القيود والتقارير
والداش بورد كلها صحيحة.

> **مهم:** لازم تشغّله على **كمبيوتر** (لابتوب ويندوز أو ماك)، مو على الجوال.
> السبب: السكربت يحتاج Node.js وهو لا يعمل على الأندرويد مباشرة.

---

## ما الذي تحتاجه (مرة واحدة فقط)

1. **كمبيوتر** فيه اتصال إنترنت.
2. **Node.js** — حمّله من https://nodejs.org (اختر النسخة LTS، التثبيت "التالي،
   التالي، تم").
3. **رابط قاعدة بياناتك** (DATABASE_URL) — نفس الرابط من Neon الذي يبدأ بـ
   `postgresql://...neon.tech/...`

---

## خطوات التشغيل

### 1) نزّل الكود

افتح **Terminal** (على الماك) أو **PowerShell** (على ويندوز) وانسخ هذه الأوامر
سطراً سطراً:

```bash
git clone https://github.com/mohamedabd11/masarat-erp.git
cd masarat-erp
git checkout claude/mvp-first-personal-projects-12wmzq
```

### 2) ثبّت المكتبات

```bash
npm install -g pnpm
pnpm install
```

### 3) شغّل بذور البيانات

انسخ السطر التالي **كاملاً** وضع مكان `ضع_الرابط_هنا` رابط قاعدة بياناتك من Neon:

```bash
DATABASE_URL="ضع_الرابط_هنا" pnpm --filter @masarat/web exec tsx scripts/seed-demo.ts
```

مثال (الرابط وهمي للتوضيح فقط):

```bash
DATABASE_URL="postgresql://neondb_owner:كلمة_السر@ep-xxx.neon.tech/neondb?sslmode=require" pnpm --filter @masarat/web exec tsx scripts/seed-demo.ts
```

## ماذا يحدث بعد التشغيل؟

السكربت سيطبع تقدّمه ثم رسالة نجاح. افتح النظام وسجّل دخول ببريدك
`zoolsamet4@gmail.com` وستجد:

| السيناريو | ما تتحقق منه |
|---|---|
| فاتورة طيران (وكيل، مدفوعة كاملة) | الإيراد = العمولة فقط، لا كامل قيمة التذكرة |
| باقة سياحية (أصيل، قسطين) | الإيراد = كامل القيمة، ودفعة تنقص الرصيد المستحق |
| عمرة (إيراد مؤجل) | يُسجَّل في حساب 3201 إيراد مؤجل، لا 4100 |
| حجز بدون دفع | يظهر كمستحق على العميل دون قيد إيراد |
| استرداد جزئي (إشعار دائن) | عكس صحيح للضريبة والإيراد |
| دفعة مورد بالريال | يُنقص رصيد المورد وحساب 2000 بنفس المبلغ |
| دفعة مورد بعملة أجنبية | فرق الصرف يذهب للأرباح/الخسائر (5900/4900) لا لرصيد المورد |

## أسئلة شائعة

**هل يكرّر البيانات لو شغّلته مرتين؟** لا. السكربت آمن — لو أعدته لا يضيف نسخاً
مكررة ولا يحذف شيئاً (معرّفات ثابتة + ON CONFLICT DO NOTHING).

**هل يحذف بياناتي الحالية؟** لا، أبداً. لا يلمس أي سجل مالي موجود.

**ظهر خطأ؟** انسخ نص الخطأ كاملاً وأرسله لي وأساعدك فوراً.
